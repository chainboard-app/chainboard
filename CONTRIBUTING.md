\# Contributing to ChainBoard



Thank you for your interest in contributing to ChainBoard! 🎉

We welcome contributions from everyone.



\## Getting Started



\### Prerequisites

\- Node.js v18+

\- pnpm

\- PostgreSQL

\- Git



\### Setup

1\. Fork the repository

2\. Clone your fork:

```bash

&#x20;  git clone https://github.com/YOUR\_USERNAME/chainboard.git

&#x20;  cd chainboard

```

3\. Install dependencies:

```bash

&#x20;  pnpm install

```

4\. Copy the environment file:

```bash

&#x20;  cp .env.example .env

```

5\. Fill in your environment variables in `.env`



\## How to Contribute



\### 1. Find an Issue

\- Browse \[open issues](https://github.com/chainboard-app/chainboard/issues)

\- Look for issues tagged `good first issue` if you're new

\- Comment on the issue to let maintainers know you're working on it



\### 2. Create a Branch

```bash

git checkout -b feature/your-feature-name

```



\### 3. Make Your Changes

\- Write clean, readable code

\- Follow the existing code style

\- Add tests where applicable



\### 4. Commit Your Changes

We follow conventional commits:

```bash

git commit -m "feat: add your feature"

git commit -m "fix: fix a bug"

git commit -m "docs: update documentation"

git commit -m "chore: update dependencies"

```



\### 5. Push and Open a PR

```bash

git push origin feature/your-feature-name

```

Then open a Pull Request on GitHub.



\## Code Style

\- TypeScript for all frontend and backend code

\- ESLint + Prettier for formatting

\- Run `pnpm run lint` before submitting



\## Branch Naming

\- `feature/` - new features

\- `fix/` - bug fixes

\- `docs/` - documentation

\- `chore/` - maintenance



\## Response Time

Maintainers will review PRs within \*\*48 hours\*\*.



\## Need Help?

Open a \[GitHub Discussion](https://github.com/chainboard-app/chainboard/discussions)

or comment on the relevant issue.



\## Code of Conduct

Please be respectful and inclusive in all interactions.
